## What Are Agent-to-Agent Payments and Why Do They Matter for Crypto?

Agent-to-agent payments eliminate human intermediaries from API transactions entirely. An AI agent requests a resource, negotiates price, settles payment, and receives data — all within a single HTTP round-trip. The x402 protocol uses the dormant HTTP 402 status code to embed on-chain payment instructions directly into API responses, while Stripe's Meter & Pay Protocol (MPP) extends traditional payment infrastructure into the same territory from the fiat side.

The convergence matters for crypto ecosystems because on-chain settlement is the native path for x402 transactions. Base, as Coinbase's L2, sits at the center of this flow. ## How Does x402 Compare to Stripe MPP for AI API Monetization?

The two protocols target the same use case from opposite ends. x402 is crypto-native: an AI agent receives a 402 response, reads the payment schema, signs an on-chain transaction (typically on Base via USDC), and the server verifies settlement before returning the payload. No accounts, no API keys, no invoices. Stripe MPP wraps the same flow in traditional fintech rails — metered usage, automated billing, card-on-file settlement.

For velocity watchers, the distinction is structural. x402 transactions generate on-chain activity that BaseRadar can measure in real time. Stripe MPP transactions stay off-chain. ## FAQ

### What is x402 and how does it enable agent-to-agent payments?

x402 is a protocol that uses the HTTP 402 Payment Required status code to embed on-chain payment instructions into API responses. When an AI agent requests a resource, the server responds with a 402 containing payment details. The agent signs an on-chain transaction, typically using USDC on Base, and the server verifies settlement before returning the data. This creates a fully autonomous payment flow with no human intervention.

### How is Stripe MPP different from x402 for AI API monetization?

Stripe's Meter & Pay Protocol handles the same metered API billing use case but through traditional payment rails. MPP uses card-on-file settlement and Stripe's existing billing infrastructure rather than on-chain transactions. The key difference for ecosystem analysis is that x402 payments generate on-chain velocity data that platforms like BaseRadar can track, while Stripe MPP transactions remain off-chain and invisible to velocity scoring.
