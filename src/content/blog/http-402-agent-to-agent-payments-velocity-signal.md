## FAQ

### What is an HTTP 402 payment in crypto?

HTTP 402 is a status code meaning "Payment Required." In the x402 protocol, when an AI agent requests a paid resource, the server responds with 402 and payment instructions. The agent settles on-chain automatically and receives the resource. This mechanism now extends to agent-to-agent payments, where AI agents pay each other directly for completed subtasks and inference results.

### Which blockchain has the strongest agent payment velocity signals in March 2026?

As of March 25, 2026, Base leads with a 25.7 average velocity score across 28 tracked tokens and the only RISING signal in the dataset — PLAY BASE at velocity 60. Solana averages 20.0 across 21 tokens with zero RISING signals. Base's higher velocity, broader token coverage, and active RISING transition indicate it is the primary settlement layer for agent payment flows.

### How can velocity data detect agent-to-agent payments before they show up in price?

Agent-to-agent payments are utility transactions — an AI agent paying another for compute or data does not create buy pressure that moves price. But these transactions do increase velocity, which measures transaction frequency relative to a token's baseline. PLAY BASE at velocity 60 with only $1.5K in 24-hour volume shows velocity running 40x ahead of what proportional human trading would produce, a signature that appears in velocity data days or weeks before any price impact materializes.

### What is the difference between single-agent and multi-agent payment velocity patterns?

Single-agent payments create regular, predictable velocity increases — one sender paying one receiver repeatedly. Multi-agent payment chains produce higher-frequency velocity with greater variance, as an orchestrator agent's payment fragments across multiple specialist agents using different settlement tokens. This is why ecosystem-wide velocity distribution matters more than individual token readings when tracking agent payment adoption.
